## Randomization Functions

**XP per Level:**
```kotlin
fun getXPForLevel(level: Int): Int {
    val baseXP = 65.0 * Math.pow(level.toDouble(), 1.1)
    val variance = 1.0 + (Math.random() * 0.1 - 0.05) // ±5%
    return (baseXP * variance).toInt()
}
```

**Points per Ability:**
```kotlin
fun getPointsForAbility(tier: Tier): Int {
    val basePoints = when (tier) {
        Tier.LIFE -> 15.0
        Tier.BEGINNER -> 22.5
        Tier.NOVICE -> 30.0
        Tier.HARD -> 45.0
        Tier.EXPERT -> 60.0
        Tier.MASTER -> 75.0
    }
    val variance = 1.0 + (Math.random() * 0.2 - 0.1) // ±10%
    return (basePoints * variance).toInt()
}
```

**Abilities per Skill:**
```kotlin
fun getAbilitiesPerSkill(tier: Tier): Int {
    val baseAbilities = when (tier) {
        Tier.LIFE -> 9.0
        Tier.BEGINNER -> 11.0
        Tier.NOVICE -> 13.5
        Tier.HARD -> 16.5
        Tier.EXPERT -> 20.0
        Tier.MASTER -> 25.0
    }
    val variance = (Math.random() * 4).toInt() // 0-3 extra
    return baseAbilities.toInt() + variance
}
```

## Implementation Notes

1. **Data-Driven Approach**: All tier data should be stored in JSON or Kotlin data classes for easy modification
2. **Progression Scaling**: XP requirements grow exponentially (Level^1.1) for natural progression
3. **Randomization**: Apply at game start or when generating new skill trees
4. **Expandability**: New tiers can be added beyond Level 200 easily
5. **Balance**: Test with actual gameplay to fine-tune values
6. **XP Loss**: When selling abilities, lose full XP gained from that ability
7. **Penalty**: Regaining lost levels requires 10% extra XP
